问题 5： ConfigMap subPath 挂载不支持热更新  
现象：修改  ConfigMap 后， Pod 内文件不自动同步。
原因：Kubernetes 已知限制 —— 使用  subPath 挂载时， ConfigMap 更新不会自动传播到  Pod 。

原因：.github/workflows/deploy.yml 引用了  ${{ secrets.SWR_AK }} 和 ${{ secrets.SWR_SK }}，但
仓库  Settings →  Secrets 中尚未添加这两个值。
解决方法 ：GitHub 仓库  →  Settings →  Secrets and variables →  Actions →  New repository secret ，分别添加  
SWR_AK（华为云  AK ）、SWR_SK（SWR 临时登录密钥）、 KUBE_CONFIG（base64 编码的  kubeconfig ）。
问题 16： SWR 登录报  Authenticate Error  
现象：Secrets 配置后重新触发， Login to SWR 步骤仍报  Authenticate Error。
原因：SWR_SK 填写的是  IAM 账号的永久  SK ，而华为云  SWR 的  docker login 需要使用  SWR 临时登录密钥 （24 
小时有效），两者不同。
解决方法 ：进入  SWR 控制台  →  右上角 " 登录指令 " →  复制其中  -p 后面的临时  Token ，更新  GitHub Secrets 中
的 SWR_SK。
